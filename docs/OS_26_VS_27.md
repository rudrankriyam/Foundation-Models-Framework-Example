# OS 26 vs OS 27

This comparison separates framework capability changes from benchmark results.
An OS result still requires the same physical device, fixtures, configuration,
and AppBench commit.

## OS 26 Baseline

OS 26 introduced developer access to the on-device Apple foundation model with:

- `SystemLanguageModel` and `LanguageModelSession`.
- Text prompting and cumulative streaming snapshots.
- Guided generation with `@Generable`, guides, and dynamic schemas.
- Custom tool calling.
- On-device execution that can work offline with no per-request cloud charge.

OS 26.4 added public `SystemLanguageModel.tokenCount(for:)` APIs. AppBench uses
them for exact on-device input, schema, response, and first-snapshot token
counts. OS 26.0 through 26.3 use a clearly marked estimate.

## OS 27 Additions

Apple's
[WWDC26 framework overview](https://developer.apple.com/videos/play/wwdc2026/241/)
describes the 2027 platform release as a broader model and agent ecosystem.
The Xcode 27 SDK adds or expands:

- A newer on-device model with improved instruction following and tool use.
- Image input and multimodal prompting.
- `PrivateCloudComputeLanguageModel`.
- A shared language-model abstraction for composing on-device, PCC, partner,
  and open-source model integrations.
- Explicit tool-calling control and richer agentic context management.
- Dynamic profiles, system tools, and built-in semantic-search integration.
- The Evaluations framework and Xcode evaluation reports.
- The `fm` command-line tool and a Python SDK on macOS.
- Open-source framework components and Foundation Models utilities.

These API additions make OS 27 a larger product surface, not merely a faster
version of OS 26.

## What AppBench Should Compare

### Same-device on-device comparison

Run the same suite on the same Mac or iPhone before and after the OS update.
Compare:

- Prompt pass and constraint score.
- Failure types.
- Median and p90 time to first streamed output.
- Median and p90 decode throughput.
- Maximum stream-update gap.
- Thermal and power state.

This captures model, runtime, scheduler, and framework changes together. It
cannot isolate new model weights from OS implementation changes.

### On-device versus PCC

This is an execution-path comparison, not a device-speed comparison. PCC adds
network, queueing, server inference, quota, and service availability. Compare
end-to-end usefulness and latency, while keeping PCC reasoning level and context
configuration fixed.

### New OS 27-only suites

Future suites should cover:

- Image-grounded extraction and question answering.
- Multi-tool planning, argument accuracy, and recovery.
- Long-context behavior near the on-device 4K and PCC 32K limits.
- Reasoning-level quality and latency tradeoffs.
- Shared-model composition and fallback behavior.

These should be separate suites so the OS 26 common corpus remains directly
comparable.

## Interpretation Rule

Do not call an OS faster or better from one run. Publish the raw build number,
at least five repetitions, distribution statistics, failures, and device state.
PCC results also need a timestamp because the server can change without a client
OS update.
